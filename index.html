Delete Files Automator Action

Apple's Automator is a handy application for basic OSX automation. One thing it doesn't ship with, however, is the ability to delete files. It can move items to the trash, but it can't empty the trash without a little help. Delete Files.action makes file deletion possible, and provides the following:

  • Standard (delete-only) and secure deletion.
  • Recursive deletion.
  • The ability to delete files only, while leaving directory structures in place.
  • Force deletion of read-only files.
  • Optional confirmation prompting, with custom confirmation text.
  • Some example workflows to get you started.

I first released this back in September of 2005; the current version is the same as what was released then, but should play more nicely with modern OSX installs. If you have any trouble with it, please contact me.

JMPrevNextAccessoryView

JMPrevNextAccessoryView is a simple inputAccessoryView for UITextFields and UITextViews which provides serial navigation functionality similar to that which Safari provides when navigating form elements. It's easy to integrate, flexible, and its appearance is customizable for each associated field.

objcutils

Various shims and jigs I've created while writing objective-c code, which don't merit their own distros. Mostly (actually, currently only) categories, including:

  • NSArray+ArrayUtils: Category on NSArray which provides static and instance methods which, similar to Python list comprehensions, allow one array to be transformed into another, with a function (block) applied to each element in turn.
  • NSArray+StringUtils: Category on NSString with the following methods:
    - (BOOL) hasSubString:
    Check whether subString is contained within an NSString instance.
    - (BOOL) startsWithDigit
    Hopefully, self explanatory.
    + (NSString *) stringByJoiningArray:withDelimiterString:;
    Easily create an NSString by joining the string representations of elements of an NSArray. Yes, you could argue that this should be a category on NSArray instead. Go for it.
    + (NSString *) stringByJoiningDict:withKeyValueDelimiterString:withPairDelimiterString:
    Similar to +stringByJoiningArray:withDelimiterString:, this makes it easy to join the key/value pairs of a dictionary into a string.
    + (NSString*)md5HexDigestString:
    Static factory method to create an MD5 digest of the given NSString.
    - (NSString *)md5HexDigest
    Instance method on NSString to create an MD5 digest.
